多线程编程

多线程相关概念

  1. 非正式来说，进程是一个执行的程序。一个进程不仅仅是程序代码（文本部分）。它还包括当前进行的活动：
    (1)程序计数器的值（用来指示下一条需要执行的指令）
    (2)对应于进程的处理器寄存器的内容
    (3)通常还有一个进程堆栈，包括临时数据（函数参数，返回地址，局部变量）
    (4)通常还有数据段，包括全局变量
    (5)可能还有一个堆，用来在运行时动态分配内存
  2. 线程是CPU利用的基本单元，它包含一个线程ID，一个程序计数器，一组寄存器，和一个堆栈。它与其它的线程共享属于同一 进程的代码块，数据块和其它的操作系统资源，例如打开的文件，信号等。

    多线程编程（基于PTHREAD)

    线程创建

    #include <pthread.h>
int pthread_create(pthread_t *restrict tidp, const pthread_attr_t *restrict attr,
 void *(*start_rtn)(void *), void *restrict arg);
    说明：
    (1) 返回0表示调用成功，否则返回一个错误码标识错误
    (2) tidp所指内存用来存储新创建线程的ID
    (3) attr表示线程的各种属性，可以设置为nullptr（NULL）表示默认属性
    (4) 新建线程从执行start_rtn函数开始
    (5) arg表示传入新建线程的参数

    线程结束

    方式1：
    正常从start_rtn函数退出，返回值就是线程退出的值。
    方式2：
    被同一进程中的其它线程取消，使用函数是pthread_cancel。 
    #include <pthread.h>
int pthread_cancel(pthread_t tid);
    说明：
    (1) 返回0表示调用成功，否则返回一个错误码标识错误 (2) tid表示需要取消的线程ID 方式3： 线程调用pthread_exit。
    #include <pthread.h>
void pthread_exit(void *rval_ptr);
    说明：
    (1) 无返回值
    (2) 参数为一个无类型的指针。
    (3) 传入的指针，可以被同一进程中对该线程调用pthread_join的其它线程捕获。
    #include <pthread.h>
int pthread_join(pthread_t thread, void **rval_ptr);
    说明：
    (1) 返回0表示调用成功，否则返回一个错误码标识错误
    (2) rval_ptr表示线程退出的值

一个关联函数： 

#include <pthread.h>
int pthread_detach(pthread_t tid);

说明：默认情况下，一个线程的结束状态会被保存，直到我们调用pthread_join函数。如果我们对这个线程调用 pthread_detach函数，那么这个线程的底层存储信息在线程结束的时候就会被回收，我们不能对这个线程调用 pthread_join函数，因为这样做会导致未定义行为。

常用多线程加锁机制

  1. pthread_mutex_t
    mutex是一种互斥锁，我们在访问保护数据前，获取它，然后在访问完保护数据之后，释放它。如果有线程获得了这个 mutex，那么其它试图获取这个mutex的线程会被阻塞，直到这个mutex被释放，然后所有等待这个锁的线程中，获得这个 锁的线程继续执行，其它线程再次进入等待。mutex在PTHREAD中用pthread_mutex_t来表示。 在我们使用 这个mutex之前，需要先将它初始化为PTHREAD_MUTEX_INITIALIZER（对静态分配的mutexes），或者对其调用 pthread_mutex_init。如果我们动态方式分配了锁(例如调用malloc)，那么我们需要在释放内存之前调用 pthread_mutex_destroy。
    #include <pthread.h>
int pthread_mutex_init(pthread_mutex_t *restrict mutex,
                     const pthread_mutexattr_t *restrict attr);
int pthread_mutex_destroy(pthread_mutex_t *mutex);
    说明：
    (1) 两个函数都是返回0表示调用成功，否则返回一个错误码标识错误
    (2) mutex表示pthread_mutex_t对象指针
    (3) attr表示锁的属性，可以设置为nullptr(NULL)表示默认属性

以下表示加锁和移出锁的函数：

#include <pthread.h>
int pthread_mutex_lock(pthread_mutex_t *mutex);
int pthread_mutex_trylock(pthread_mutex_t *mutex);
int pthread_mutex_unlock(pthread_mutex_t *mutex);

说明：
(1) 以上函数返回0表示调用成功，否则返回一个错误码标识错误
(2) mutex表示pthread_mutex_t指针

  1. mutex的超时函数
    pthread_mutex_timedlock用来设置一个超时时间段（时刻），如果时间段用完了，或者时刻已经到了，那么就算无法获取锁，函数 调用也会结束，并返回相应的错误码。

    #include <pthread.h>
#include <time.h>
int pthread_mutex_timedlock(pthread_mutex_t *restrict mutex,
                         const struct timespec *restrict tsptr);

    说明：
    (1) 返回0表示调用成功，否则返回一个错误码标识错误
    (2) mutex表示pthread_mutex_t指针
    (3) tsptr表示超时相关的时间信息

  2. pthread_rwlock_t rwlock类似于mutex。rwlock有三种状态，1)未加所 2)加了读锁 3) 加了写锁。只有一个线程可以获取同一个读写锁的 写锁状态，但是可以有很多个线程可以获取同一个读写锁的读锁状态。如果线程1获取了读写锁A的写锁，那么线程2不能 获取锁A的读锁或者写锁。如果线程1获取的是读锁，那么线程2可以获取锁A的读锁，但是不能获取锁A的写锁。读写锁 适合于读远大于写的情况下。

    #include <pthread.h>
int pthread_rwlock_init(pthread_rwlock_t *restrict rwlock,
                     const pthread_rwlockattr_t *restrict attr);
int pthread_rwlock_destroy(pthread_wrlock_t *rwlock);

    说明：和mutex函数类似，不过操作的是读写锁。静态的读写锁可以通过PTHREAD_RWLOCK_INITIALIZER初始化。

#include <pthread.h>
int pthread_rwlock_rdlock(pthread_rwlock_t *rwlock);
int pthread_rwlock_wrlock(pthread_rwlock_t *rwlock);
int pthread_rwlock_unlock(pthread_rwlock_t *rwlock);
int pthread_rwlock_tryrdlock(pthread_rwlock_t *rwlock);
int pthread_rwlock_trywrlock(pthread_rwlock_t *rwlock);

说明：和mutex函数类似，不过这里涉及读锁和写锁，其中有rd的表示获取的是读锁，有wr的表示获取的是写锁。

  1. rwlock的超时函数

    #include <pthread.h>
#include <time.h>
int pthread_rwlock_timedrdlock(pthread_rwlock_t *restrict rwlock,
                             const struct timespec *restrict tsptr);
int pthread_rwlock_timedwrlock(pthread_rwlock_t *restrict rwlock,
                             const struct timespec *restrict tsptr);

    说明：和mutex的超时函数类似，不过这里涉及读锁和写锁，其中有rd的表示获取的是读锁，有wr的表示获取的是写锁。

  2. 条件变量
    条件变量是一种用来实现线程进行交会的同步机制。借助于mutex，条件变量可以允许线程以一种线程安全的方式等待任意条件 发生。条件通过mutex来保证安全访问。一个线程必须先获得锁，然后再改变条件状态。

    #include <pthread.h>
int pthread_cond_init(pthread_cond_t *restrict cond, 
                   const pthread_condattr_t *restrict attr);
int pthread_cond_destroy(pthread_cond_t *cond);

    说明：和mutex类似，不过是初始化条件变量，静态的条件变量可以通过PTHREAD_COND_INITIALIZER初始化。

条件变量和锁有所不同，条件变量的函数主要是等待和唤醒。

#include <pthread.h>
int pthread_cond_wait(pthread_cond_t *restrict cond,
                      pthread_mutex_t *restrict mutex);
int pthread_cond_timedwait(pthread_cond_t *restrict cond,
                           pthread_mutex_t *restrict mutex,
                           const struct timespec *restrict tsptr);

说明：
(1) 以上函数都是返回0表示调用成功，否则返回一个错误码表示错误原因。
(2) 都是进入等待，等待其他线程调用signal相关函数，将其唤醒，以上函数都需要在加锁的条件下调用，然后 将锁的指针作为mutex的实参。 (3) 在结束等待之后，都会重新获得锁，然后处理相关逻辑。 

#include <pthread.h>
int pthread_cond_signal(pthread_cond_t* cond);
int pthread_cond_broadcast(pthread_cond_t *cond);

说明：
(1) 以上函数都是返回0表示调用成功，否则返回一个错误码表示错误原因。
(2) pthread_cond_signal唤醒至少一个等待该条件变量的线程，而pthread_cond_broadcast将会唤醒所有等待这个条件变量的线程。

  1. Spinlock 说明：通过轮训的方式来获取锁的一种同步方式。

多线程编程（以下说明基于：C++11及以上）

线程创建

方式1：使用std::async 实例：

#include <future>
#include <iostream>

int main()
{
    auto now = std::chrono::system_clock::now();

    auto ret = std::async([](int i, int j) 
    {
        std::this_thread::sleep_for(std::chrono::seconds(2));
        return i + j;
    }, 10, 20);

    std::this_thread::sleep_for(std::chrono::seconds(2));

    std::cout << "10 + 20 = " << std::flush;
    std::cout << ret.get() << std::endl;

    std::chrono::duration<double> dif = std::chrono::system_clock::now() - now;
    std::cout << "time spending: " << dif.count() << std::endl;
        return 0;
}

另外async在调用时，还可以传入std::launch。
实例：

#include <future>
#include <iostream>

int main()
{
    auto now = std::chrono::system_clock::now();

    auto ret = std::async(std::launch::deferred, [](int i, int j)
    {
        std::this_thread::sleep_for(std::chrono::seconds(2));
        return i + j;
    }, 10, 20);

    std::this_thread::sleep_for(std::chrono::seconds(2));

    std::cout << "10 + 20 = " << std::flush;
    std::cout << ret.get() << std::endl;

    std::chrono::duration<double> dif = std::chrono::system_clock::now() - now;
    std::cout << "time spending: " << dif.count() << std::endl;
    return 0;
}

运行上述两段代码，查看运行效果的不同。也可以试着将std::launch::deferred改为std::launch::async，查看运行效果。
std::async实用实例：

#include <future>
#include <iostream>
#include <numeric>
#include <vector>
#include <algorithm>

template <typename RandomIt>
int parallel_sum(RandomIt beg, RandomIt end)
{
    auto len = end - beg;
    if (len < 1000)
        return std::accumulate(beg, end, 0);

    RandomIt mid = beg + len / 2;
    auto handle = std::async(std::launch::async,
        parallel_sum<RandomIt>, mid, end);
    int sum = parallel_sum(beg, mid);
    return sum + handle.get();
}

int main()
{
    auto now = std::chrono::system_clock::now();

    std::vector<int> v(10000, 1);
    std::cout << "The sum is " << parallel_sum(v.begin(), v.end()) << '\n';

    std::chrono::duration<double> dif = std::chrono::system_clock::now() - now;
    std::cout << "time spending: " << dif.count() << std::endl;
        return 0;
}

方式2：使用std::thread
实例：

#include <future>
#include <iostream>

int main()
{
    auto now = std::chrono::system_clock::now();

    int sum = 0;
    auto ret = std::thread([&sum](int i, int j)
    {
        std::this_thread::sleep_for(std::chrono::seconds(2));
        sum = i + j;
    }, 10, 20);

    std::this_thread::sleep_for(std::chrono::seconds(2));

    std::cout << "10 + 20 = " << std::flush;
    ret.join();
    std::cout << sum << std::endl;

    std::chrono::duration<double> dif = std::chrono::system_clock::now() - now;
    std::cout << "time spending: " << dif.count() << std::endl;
        return 0;
}

说明上述两种方式使用的不同：
(1) std::async可以线程安全地带回一个返回值。

线程结束

方式1： 对于std::async方式调用的多线程，多线程的退出，依赖于future的实现（std::async会返回一个std::future）。 对于通过std::launch::deferred的方式调用，如果对于返回的future调用了wait()或者get()，将会等待传入的函数对象运行结束，如果没有调用上述函数，则传入的函数对象不会执行。 对于通过std::launch::async的方式调用，系统会等待函数执行完全。 对于通过std::launch::deferred|std::launch::async方式调用，也是不带std::launch的调用方式，实现会选取上述中的一种。

方式2： 对于std::thread方式调用的多线程，需要在结束之前调用join()或者detach()函数，否则程序调用abort函数。 如果采用join函数，则调用线程会等待传入的函数对象执行完成。如果调用detach函数，则传入的线程作为后台线程执行。

常用多线程加锁机制

(1) std::mutex
互斥锁，用来实现单一访问

1）lock() 用来加锁，如果当前可以获得，则获得；如果当前不能获得，则等待
2）try_lock() 尝试加锁，如果当前可以获得，则获得；如果当前不能获得，则返回false
3）unlock() 释放当前获得锁

实例： 

#include <future>
#include <iostream>
#include <numeric>
#include <vector>
#include <algorithm>

std::mutex g_mtx;

template <typename RandomIt>
void parallel_sum(RandomIt beg, RandomIt end, int &sum)
{
    auto len = end - beg;
    if (len < 1000)
    {
        sum = std::accumulate(beg, end, 0);
        return;
    }

    RandomIt mid = beg + len / 2;
    int half2 = 0;
    auto half2_handle = std::thread(parallel_sum<RandomIt>, mid, end, std::ref(half2));
    int half1 = 0;
    parallel_sum(beg, mid, half1);
    half2_handle.join();
    g_mtx.lock();
    sum = (half1 + half2);
    g_mtx.unlock();
}

int main()
{
    auto now = std::chrono::system_clock::now();

    std::vector<int> v(10000, 1);
    int sum = 0;
    parallel_sum(v.begin(), v.end(), std::ref(sum));
    std::cout << "The sum is " <<  sum << '\n';

    std::chrono::duration<double> dif = std::chrono::system_clock::now() - now;
    std::cout << "time spending: " << dif.count() << std::endl;
    return 0;
}

和上面使用std::async的一个实例实现相同的功能，但是使用std::async不需要显示使用加锁，而且实现的函数的逻辑更加简单，清楚。
下面使用std::lock_guard来略微优化上述代码： 

#include <future>
#include <iostream>
#include <numeric>
#include <vector>
#include <algorithm>

std::mutex g_mtx;

template <typename RandomIt>
void parallel_sum(RandomIt beg, RandomIt end, int &sum)
{
    auto len = end - beg;
    if (len < 1000)
    {
        sum = std::accumulate(beg, end, 0);
        return;
    }

    RandomIt mid = beg + len / 2;
    int half2 = 0;
    auto half2_handle = std::thread(parallel_sum<RandomIt>, mid, end, std::ref(half2));
    int half1 = 0;
    parallel_sum(beg, mid, half1);
    half2_handle.join();
    {
        std::lock_guard<std::mutex> lock(g_mtx);
        sum = (half1 + half2);
    }
}

int main()
{
    auto now = std::chrono::system_clock::now();

    std::vector<int> v(10000, 1);
    int sum = 0;
    parallel_sum(v.begin(), v.end(), std::ref(sum));
    std::cout << "The sum is " <<  sum << '\n';

    std::chrono::duration<double> dif = std::chrono::system_clock::now() - now;
    std::cout << "time spending: " << dif.count() << std::endl;
    return 0;
}

上述代码，只有在parallel_sum函数中设置sum的值位置有略微的修改。

(5) 上述加锁机制的辅助类：std::lock_guard std::unique_lock 作用：采用RAII机制，方便使用各种加锁机制。

(6) std::condition_variable 条件变量，用于处理多线程之间有的线程准备条件，有的线程满足条件的情况下执行。

1) notify_one() 使其中一个调用wait系列函数的线程结束等待(需要和调用notify_one的对象是同一个)，进行判断
2) notify_all() 使所有等待该对象的线程结束等待
3) wait() 使当前线程进入等待，直到有另外的线程调用cv的notify系列函数
4) wait_for() 使当前线程进入等待，直到有另外的线程调用cv的notify系列函数，或者等待的时间段过了
5) wait_until() 使当前线程进入等待，直到有另外的线程调用cv的notify系列函数，或者等待的时间到了

实例：

#include <future>
#include <iostream>
#include <condition_variable>

constexpr int MSG_NUM = 10;
bool g_isQuit = false;
std::mutex g_mtx;
std::condition_variable g_cv;

struct msg 
{
    msg *m_next = nullptr;
    int m_value = 0;

    msg(msg* next, int value)
        : m_next(next), m_value(value)
    {

    }
};
msg *workq = nullptr;

void process_one(msg* mp)
{
    std::cout << mp->m_value << std::endl;
    delete mp;
}

void process_msg(void)
{
    msg *mp;
    for (;;) 
        {
        {
            std::unique_lock<std::mutex> lk(g_mtx);
            g_cv.wait(lk, [&]() {return workq != nullptr || g_isQuit; });

            if (g_isQuit)
            {
                break;
            }

            mp = workq;
            workq = mp->m_next;
        }

        /* now process the message mp */
        process_one(mp);
    }
}

void enqueue_msg(msg *mp)
{
    {
        std::lock_guard<std::mutex> lk(g_mtx);
        mp->m_next = workq;
        workq = mp;
    }

    g_cv.notify_one();
}

void handle_all(msg *mp)
{    
    while (mp)
    {
        auto workNode = mp;
        mp = mp->m_next;
        process_one(workNode);
    }
}

int main()
{
    std::thread t(process_msg);
    std::this_thread::sleep_for(std::chrono::milliseconds(100));

    for (int i = 0; i < MSG_NUM; ++i)
    {
        enqueue_msg(new msg{ nullptr, i });
        std::this_thread::sleep_for(std::chrono::milliseconds(50));
    }

    {
        std::lock_guard<std::mutex> lg(g_mtx);
        handle_all(workq);
        g_isQuit = true;
    }
    g_cv.notify_one();
    t.join();

        return 0;
}
