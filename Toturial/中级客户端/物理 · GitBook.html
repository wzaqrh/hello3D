物理

Unity3D 物理

Unity集成NVIDIA PhysX物理引擎作为其物理模块，并在内部维护了物理世界的创建更新销毁，碰撞体的碰撞检测，刚体更新等逻辑，并封装一系列物理相关上层接口。Physx是目前使用最为广泛的物理引擎，被很多游戏大作所采用，开发者可以通过物理引擎高效、逼真地模拟刚体碰撞、车辆驾驶、布料、重力等物理效果，使游戏画面更加真实而生动

Unity 5.0之前使用的PhysX版本为PhysX 2.8.3，5.0之后升级到PhysX 3.3

刚体(Rigidbody)

刚体是物理引擎中重要的基本概念。刚体是指运动中和受力作用后，形状和大小不变，而且内部各点的相对位置不变的物体。绝对刚体实际上是不存在的，只是一种理想模型。这种假设模型能良好的配合碰撞检测系统，且能大幅简化模拟固体动力学所需的数学，并能在工程上能有足够的准确度。

GameObject添加Rigidbody组件之后物体将能受到物理作用的影响，物体将在物理引擎的驱动下运动，能和带有碰撞体的对象做出物理碰撞等交互并做出反映，并受到重力等因素的影响。

详细属性和方法参见官方文档:

  • Rigidbody Manual
  • ScriptReference

重要属性说明：

  • Mass：质量，单位为千克。
  • Drag：线速度阻尼参数，模拟空气阻力，默认为0。
  • Angular Drag：角速度阻尼参数，默认值为0.05。设置了Angular Drag大小之后，物体在任何方向上的旋转运动都将会受到影响。
  • Velocity：刚体线速度。
  • AngularVelocity：刚体角速度。
  • position:刚体位置，会在物理更新完成后应用到Transform.position
  • rotation:刚体旋转，会在物理更新完成后应用到Transform.rotation
  • Use Gravity：刚体是否受重力影响,模拟现实世界中的自由落体状态等。
  • Is Kinematic：开启之后物体运动将不再受刚体模拟，而是由Transform来控制，但其他刚体任然会对其有碰撞响应。常常用在玩家控制运动的主角物体等。
  • Continuous Dynamic：CCD检测，物理模拟默认以离散的时间步来模拟的，简单的每个时间步中每个刚体的位置被当成是静止的，然后碰撞检测发生与这些碰撞世界的“快照”中，对于移动速度相对其尺寸不大的的物体来说方法可行，但对于一些尺度较小但运动速度很快的物体模拟就会产生“Tunneling”的隧穿现象。CCD就是处理隧穿的方法之一。其算法一般使用迭代式计算，因此对计算效率由很大影响，需要谨慎使用，或者使用其他方法来解决，如简单的射线碰撞检测或简单碰撞体的swept test。
  • Interpolate：运动插值，由于刚体运动和渲染更新并非完全同步，比如对于相机跟随的主角物体，其物理更新和相机更新、渲染不在同一个Update逻辑中，这样可能对出现可见的抖动现象，可以开启插值来解决。
  • Constraints：约束条件用来设置物体在哪一个方向上运动或旋转将受到的限制。例如在主角控制中约束X,Z方向的旋转防止角色的倾倒。
  • sleepThreshold：刚体进入休眠状态的阈值，为了提高效率物理引擎中当刚体的运动速度小于一定阈值一段时间后，刚体将进入休眠状态以减少一些不必要的模拟计算。刚体的很多方法比如设置重力线速度角速度，addForce，addTorque都会使得刚体离开休眠状态。
  • unity 中，物体快速移动，可能会导致一帧位移过大从而穿过物体，使得碰撞器检测失效，有什么好办法吗？
  • Unity How to Fix Movement Stutter

重要方法说明：

  • AddForce：对刚体施加一个指定方向的力，作用于刚体使其发生移动。
  • AddTorque：给刚体施加一个力矩，使得刚体绕torque轴旋转
  • MovePosition：用来将刚体移动到position位置
  • MoveRotation：用来刚体旋转到rot。rot为Quaternion（四元数）会返回刚体需要旋转的角度。

经典力学：f = ma (force = mass acceleration)

通常情况下最好不要直接对刚体速度进行修改，而是通过AddForce之类的操作间接的对速度进行操作。因为物理引擎内部很多模拟计算是以“当前”速度为基准进行的，如果直接进行速度的设置，可能会导致一些计算的结果不太真实。而AddForce的话就可以将加速度等计算交由物理引擎自行处理。

FixedUpdate

物理部分的计算应该是与帧速率无关的，因此其模拟往往需要使用固定的步长，因此其计算往往放在FixedUpdate中，按照固定间隔执行。

碰撞体(Collider)

碰撞体(Collider):描述了一个物理世界中物体的形状。物理引擎基于这些形状来计算物理体的碰撞检测。碰撞只会发生在两个物体都带有Collider组件时。只有Collider组件而没有Rigidbody的GameObject自己不会响应物理行为的，可以归为静态碰撞体，适合与场景中不动的物体如地面、墙面、空气墙等。

触发器(Trigger):当Collider组件Inspect面板的中IsTrigger属性为true时，将成为触发器。与触发器产生碰撞的刚体不会造成碰撞响应，只会触发碰撞事件。 Unity按照Collider的形状和作用分为Box Collider，Capsule Collider，Sphere Collider，Mesh Collider,Wheel Collider,Terrain Collider。

车轮碰撞器（Wheel Collider）是一种特殊的碰撞器，它并不是特指某一种碰撞形状，而是一种包含碰撞检测、车轮物理引擎和基于滑动的轮胎摩擦的物理模型。很多车辆物理的插件都是基于Wheel Collider开发。如Edy’s Vehicle Physics

物理管理器

Unity Manual - PhysicsManager

物理管理器通过Edit→Project Settings→Physics来打开

  • Gravity（重力）：在Unity中默认的重力的方向为Y轴的负方向，大小为9.81N。
  • Default Material（默认材质）：全局管理器中默认没有使用任何物理材质，在其中开发人员可以为其添加物理材质。
  • Bounce Threshold（反弹阈值）：在两个相互碰撞的刚体间的相对速度小于阈值时，就不会再进行反弹计算，这样会有效的减少模拟物理过程中物体的抖动与物理计算。
  • Raycasts Hit Triggers（射线检测触发器）：Unity集成开发环境的物理引擎中3D射线拾取功能需要和碰撞器相互配合使用，即射线命中碰撞器之后会返回命中信息，被开发人员捕获后可以实现特定的功能。
  • Layer Collision Matrix（层碰撞矩阵）：Unity集成开发环境中可开发人员可以对场景中的物体进行分层，将不同功能或类型的物体区分开来。在物理管理器中，可以使用层碰撞矩阵来设置不同层的物体间的碰撞计算。两个层的交叉处就是设置碰撞检测的标志位。

Physics类基本函数

Raycasts：

射线检测，从起点朝某一方向发射一条射线，来检测射线与场景中Collider相交性。常用来进行鼠标点选，触地检测等功能。相关API：

  • Ray Camera.main.ScreenPointToRay(Vector3 pos) 返回一条射线Ray从摄像机到屏幕指定一个点
  • Ray Camera.main.ViewportPointToRay(Vector3 pos) 返回一条射线Ray从摄像机到视口（视口之外无效）指定一个点
  • Ray 射线类
  • RaycastHit 射线投射碰撞信息
  • bool Physics.Raycast(Vector3 origin, Vector3 direction, float distance, int layerMask) 当射线投射与任何碰撞器交叉时为真，否则为假。
  • bool Physics.Raycast(Ray ray, Vector3 direction, RaycastHit out hit, float distance, int layerMask) 在场景中投下可与所有碰撞器碰撞的一条射线，并返回碰撞的详细信息。
  • bool Physics.Raycast(Ray ray, float distance, int layerMask) 当射线投射与任何碰撞器交叉时为真，否则为假。
  • bool Physics.Raycast(Vector3 origin, Vector3 direction, RaycastHit out hit,float distance, int layerMask)当射线投射与任何碰撞器交叉时为真，否则为假。
  • RaycastHit[] RaycastAll(Ray ray, float distance, int layerMask) 投射一条光线并返回所有碰撞，也就是投射光线并返回一个RaycastHit[]结构体。

Overlaps：

检测场景中与某一类型碰撞几何体相交的Collider，常用来进行范围相交判定如攻击判定等场景。相关API：

  • Physics.OverlapBox/OverlapBoxNonAlloc：检测几何体形状为Box
  • Physics.OverlapCapsule/OverlapCapsuleNonAlloc:检测几何体形状为Capsule
  • Physics.OverlapSphere/OverlapSphereNonAlloc:检测几何体相撞为Sphere

Sweeps：

类似RayCast，投射的不是射线而是给定的形状，常用在计算检测运动物体前后帧之间发生的碰撞行为。相关API：

  • Physics.BoxCast/BoxCastAll/BoxCastNonAlloc:投射体为Box
  • Physics.CapsuleCast/CapsuleCastAll/CapsuleCastNonAlloc:投射体为Capsule
  • Physics.SphereCast/SphereCastAll/SphereCastNonAlloc:投射体为Sphere

Penetration Depth:

计算两个给定的Collider在特定的位置旋转上的穿插渗透情况，常用在实现自定义的角色控制器上。相关API：

  • Physics.ComputePenetration：计算两个Collider穿插情况
  • Physics.ClosestPoint：计算特定位置离Collider上最近点
