动画 - UV 动画、顶点动画的原理与应用

《UV动画、顶点动画的原理与应用》

3D动画的两种常用实现方式的原理

3D模型的顶点格式中一般包含两部分信息：贴图数据，顶点数据。贴图数据一般是通过UV进行描述的，而顶点数据则通过三维坐标进行描述。因此，不难得出3D中最基础的两种动画实现方式，UV动画与顶点动画。

UV动画——本质通过改变贴图

UV动画，主要用于序列帧、模拟无限重复的背景让模型顶点对应的UV随时间变化，使其对应纹理像素不断随时间变化但是无法达到让物体“变形”的效果。UV动画是一种常用的渲染技巧，经常用来描述水的流动、霓虹灯的闪烁等。实现的原理就是动态修改贴图的UV坐标，使物体表面产生变化。

说到UV动画就不得不多说一点贴图了。我们知道，贴图是覆盖在模型表面用来描述模型表面细节的2D图片，那么贴图是怎么覆盖到模型上的呢？我们知道模型是由很多顶点组成的，这些顶点，三个一个组组成了若干三角形，正是这些三角形构成了模型的表面(三个点确定一个面嘛)，这样的三个点每一个都有一组贴图坐标，表示贴图上的某个点，这样三组贴图坐标就可以贴图上的一个区域，这个区域就是将来贴在这个三角形的部分。这里的贴图坐标也叫UV坐标，是一组[0,1]之间的二维向量，以贴图左上角为(0,0)点，水平方向向右，水平方向向下增长。

顶点动画——本质通过改变顶点坐标

顶点动画，原理和UV动画有相同的地方，但是顶点动画可以让物体动起来的同时，达到变形的效果其主要原理是在 顶点着色器中对模型的顶点进行偏移，再将偏移后的顶点位置输出到片元着色器中。与改变mesh顶点不同的是，在gpu中更改顶点信息的效率远高于在cpu中改变mesh的数据。这种渲染技巧普遍用于模拟自然界物体波动，如水流、草地、火焰等效果。

UV动画+顶点动画

两种动画的同时使用，在粒子特效系统中大量存在。很多特效都是控制基础面片在改变UV的同时拉升、缩放顶点。当这样的基础片面（粒子）很多时，表现出的效果是惊人的。

变形目标动画（Morph Target Animation）——本质还是在控制顶点

仅制作相对较少的固定极端姿势（Extream Pose）。在运行时把两个或以上的这些姿势混合，就能生成动画。变形目标技术通常用于面部动画（Facial Animation），因为人脸具有非常复杂的解剖结构。

草块的产生（顶点生成）

​ 在每个草块刷新时，遍历所属该草块的采样点（通过预设的高度值和密度值）。在每个采样点圆形范围（配置radius，radius的大小会决定与相邻采样点交叉范围，越小则草地越规则）内生成相应数量（密度因数 配置数量 渲染质量等级因数 + 随机值）个不同大小（密度因数 = clamp(密度/密度百分比，配置草体最小值，float.MaxValue))的草个体mesh。这种大小设置的原因是在草地与非草地边缘，草体的大小会随权重图通道过度而变化，自然的形成地貌分割。设置的最小值则为了不让草体过小。

​ 上面的过程可以获得草体的生成数量、大小。接下来生成mesh的同时我们则在采样点的radius范围内随机草体位置，并加上采样点的世界坐标，来获得草体mesh生成时所需的世界坐标偏移量（worldOffset）。生成草体的mesh时我们以quad为例：

​ 草体4个顶点在采样点坐标系的坐标应该是：

​ (-width,height) (width,height)

​ (-width,0) (width,0)

​ 我们需要对草体的方向进行随机，所以我们对向量Dir = -width进行旋转。形成随机方向：

NewDir = Vector3.SlerpUnclamped(new Vector3(-width,0,0), new Vector3(width,0,0),Random.Range(-1f,1f));

​ 则新的顶点世界坐标为：

​ a: v3(-dir.x,height,-dir.z) + worldOffset b: v3(dir.x,height,dir.z) + worldOffset

​ c: v3(-dir.x, 0f, -dir.z) + worldOffset d: v3(dir.x,0f,dir.z) + worldOffset

​ 1、按照顺序（c,a,b,d）将顶点坐标添加到mesh的顶点List中。

​ 2、给mesh添加顶点索引(m_Mesh.SetIndices(indexList.ToArray(),MeshTopology.Quads,0);)。使用MeshTopology.Quads类型的拓扑结构可以不用自己计算索引顺序，这也是上面添加顶点时候使用顺时针顺序（c,a,b,d）的原因。

​ 3、添加草体顶点法线方向，这里默认可以用（Vector3.up）。

​ 4、添加顶点色（从预设置的颜色数组中选取一种颜色，如果设置了特殊区域颜色，可以在这里进行区域判定，是否使用特殊颜色）。这里我们设置草体颜色的时候alpha通道没有用，那么正好存储一个随机种子在alpha上，这个种子不参与颜色计算，用来在shader计算中当做随机值使用（风力作用大小，力场作用大小等需要随机数的地方，来减小gpu开销）。每个草体的世界偏移量都是随机的，那么我们就用这个特性来生成草体种子：

Color c = GetColor(SpecRange(worldPos));
float alpha = vertsList[i].x * 100.0f + vertsList[i].z;
color.a = alpha – Mathf.Floor(alpha);
Color32 c32 = c;
colors32.Add(c32);
m_Mesh.color32 = colors32.ToArray();

5、设置顶点uv，由于面片为quad，所以顺序添加（0,0）、（0,1）、（1,1）、（1,0）即可。

6、将新的mesh设置给草块的MeshFilter即可。（记得Destroy原来的meshFilter.mesh）

麦浪效果（顶点动画的控制方法）

思路：在顶点着色器中对顶点的世界坐标进行偏移，制造风吹的效果。

​ 提到“浪”则需要函数的波形来模拟该现象，由于没有海浪那么复杂。那么我们可以通过一个sin函数来解决这个问题，如果设备性能允许，也可以考虑用更复杂的波形来模拟，在此只介绍一个简单的sin函数。

​ 经验公式:

//取得顶点的世界坐标
float4 worldPos = mul(unity_ObjectToWorld, v.vertex);
//计算x轴偏移值
float xOffset = (pow(sin(_Time.y * 1.5f + pos.x / 6.0f), 3) / 1.5f – 0.3f) * _GrassWindFactor                    * uv.y * uv.y;
//改变y轴高度，避免草被拉长
float yOffset = sqrt(y * y – xOffset * xOffset);
//计算位移后结果
float3 p = float3(xOffset, (yOffset – y) * 0.75f , 0);
//转换到齐次坐标
o.pos = mul(UNITY_MATRIX_VP,float4(p,1));

​ 首选要计算顶点的xOffset，sin基本波形的基础上，由时间_Time.y驱动，顶点的坐标(pos.x)影响函数的周期内阶段，这样同一时间内不同位置的草处于不同的波形阶段。3次方是为了将函数图像变得陡峭的同时又不影响朝向。除以1.5使函数范围变小并减去0.3使得函数图像整体向y轴反方向偏移，造成草在摆动过程中的不对称，最后乘以风力因数和uv.y^2，这样做要建立在草的贴图是由下到上的基础上，意思是越靠近草的顶端则风力影响越大，平方处理使得影响非线性。最后根据勾股定理求得yOffset的高度变化，保持草整体长度不变。

草地生长过渡（顶点生成的细节控制）

​ 向shader中传入摄像机或者主角的世界坐标，计算草与玩家距离后在接近可视边缘一定范围内做草地高度的差值运算，如下图所示绿色为正常渲染的草地范围，红色为过度差值范围，红色以外为不可见范围。在红色范围内随着远离摄像机坐标，草的高度逐渐降低。这种做法是为了草在视野内的自然出现，不会闪现出来。

​ 具体做法则是设置一个过渡范围dis，dis应小于草地的边长/2。在草地的顶点渲染中计算顶点与人物（或摄像机）的距离，然后与设置的dis范围对比，如果在范围内，则差值出当前顶点的应有高度：lerp（0， 草体高度， curDis/dis）。实现草地边缘的高度过渡，营造出草地逐渐生长出来的效果。

UV动画控制细节

UV动画在渲染中有多重用途，我们这里主要使用的是帧动画，下面介绍一下使用目的。支持uv动画的功能后，草片内部的摆动以及每根草之间的交互可以由美术来完成，整体来看草原受到风的影响，同时局部范围内每根草之间还有交互，效果看起来很好，同时性能开销不大。

实现上的一点技巧就是之前我们在设置顶点颜色的时候，在alpha通道上设置了一个随机值，将这个随机值运用到帧动画里，这样每个草就会处于uv动画的不同阶段，使整片草原看起来无序且自然。

具体实现：

在草片对贴图的采样过程中，通过设置的序列帧行数、列数、频率、初始帧（通过alpha的通道的随机值生成）等因素，并由时间作为驱动计算出uv的采样区间。建议设置帧率为24，会比较流程，又不会过多的浪费贴图空间。
